**1. 문자열 (String Manipulation)**

• **인덱싱 (Indexing):**

## ◦  <font color="#c00000">0부터</font> 시작 (`s`). 
처음으로 빨간색을 입혔다. 그정도로 중요하다.
진심으로 여기서 1부터 세는 습관이 생기면 평생 간다.
저주할거다. 물론 나는 아니고 내 친구 짱구가 그러더라.

   ◦ 음수 인덱스는 뒤에서부터 셈 (`s[-1]`은 마지막 문자) 
```

def index(x):
	print(set[x])

print(index(-3))
print(index(-2))
print(index(-1))

print('--------------대칭-----------------')

print(index(0))
print(index(1))
print(index(2))
print(index(3))
#print(index(4)) -> list index out of range

```



• **슬라이싱 (Slicing):** `s[start:stop:step]` 
슬라이싱이라는 단어의 어감이 좀 그러면 이렇게 생각하자.
슬라이스 치즈. ~~있는건가..?~~ 
슬라이딩 ~~비슷한거긴한가..??~~

   ◦ `start`부터 `stop-1`까지 `step` 간격으로 가져옴.

   ◦ `s[::-1]`: 문자열 뒤집기 (자주 쓰이는 테크닉!)  <-- 이건 진짜 자주 쓴다. 진심이다. 
   ~~진짜가 아닌 마음을 난 진심이라 하기로 했다.~~ 근데 일단 난 자주 쓴다.

```
set = [1,2,3,4,5,6]

print(set[1:]) #index 1부터 끝까지
print(set[:5]) #index 0부터 5까지
print(set[4:5]) #index 4 부터 5까지
print(set[::-1]) #문자열 뒤집기
print(set[::-2]) #문자열 뒤집기 & 두 칸씩
```

**3. 알고리즘 전략 2: Approximate Solutions**

• **개념:** 정확한 답(Exact match)을 찾기 어려울 때, "충분히 좋은(good enough)" 답을 찾는 방법.

우리 모두 고등학교 2학년때 수학2에서 첫 단원이 **함수의 극한**이다.
이때 배우는 함수의 극한에서의 수렴이 이와 유사한 개념이라 이해하면 된다

• **핵심 요소:**

 `epsilon` (ϵ): 허용 오차 범위 (예: 0.01). 답이 이 범위 안에 들어오면 성공으로 간주한다

 ◦ `step` (증가량): 조금씩 값을 증가시키며 확인.

• **트레이드오프:**

    ◦ `step`이 너무 작으면? → 정확도는 높지만 속도가 느림.

    ◦ `step`이 너무 크면? → 답을 건너뛸(Skip) 위험이 있음 [Source 155].

**4. 알고리즘 전략 3: Bisection Search**

• **개념:** 탐색 범위를 절반씩 줄여가며 답을 찾는 방법 (Up/Down 게임과 동일) 

• **조건:** 데이터가 **정렬(Sorted)되어 있거나, 답의 범위가 연속적이어야 함.

• **성능:** O(logN)으로 매우 빠름. (예: 100만 개의 데이터도 20번 정도면 찾음)
