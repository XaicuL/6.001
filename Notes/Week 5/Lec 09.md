
# **Classes and Inheritance**

## **# 1. 상속(Inheritance): 계층적 구조화의 필연성**

상속은 상위 클래스(Superclass)의 데이터와 행동을 하위 클래스(Subclass)가 물려받아 
논리를 확장하는 메커니즘이다.

### **[Necessity] 상속의 필연적 등장 배경**

- **기존 방식의 모순**: 서로 유사한 객체들(Cat, Rabbit, Person)을 정의할 때마다 공통 속성(age, name)을 반복해서 코딩하는 것은 관리의 비효율과 코드 중복을 초래한다.
    
- **해결책**: 모든 동물의 공통 분모는 `Animal` 클래스에 박제하고, 하위 클래스는 자신만의 고유한 기능(예: `speak()`)만 추가하거나 기존 기능을 재정의(Override)하여 복잡성을 통제한다.

## **# 2. 정보 은닉(Information Hiding): 캡슐화의 당위성**

클래스 내부의 데이터 표현 방식은 외부로부터 보호되어야 하며, 오직 정해진 통로를 통해서만 소통해야 합니다.

### **Getter와 Setter의 엄밀성**

- **[Necessity] 유지보수의 필연성**: 외부에서 `a.age`처럼 데이터에 직접 접근하면, 나중에 내부 변수명이 `age`에서 `years`로 바뀔 때 외부의 모든 코드가 붕괴된다.
    
- **해결책**: 데이터 접근은 반드시 **Getter**(`get_age()`)와 **Setter**(`set_age()`)를 경유하도록 강제하여, 내부 구현이 변해도 외부 인터페이스는 유지되는 **데이터 추상화**를 완성한다.

## **# 3. 클래스 변수(Class Variables): 데이터 공유 메커니즘**

인스턴스 변수가 개별 객체의 상태를 담는다면, 
클래스 변수는 해당 타입의 **모든 인스턴스가 공유**하는 전역적 상태라고 이해하면 좋다.

- **[Analysis] 고유 ID(Tag) 부여 전략**:
    
    - **[Necessity]**: 생성되는 모든 객체에 중복되지 않는 고유 번호를 부여해야 할 때 
      클래스 변수를 활용한다
        
    - **메커니즘**: `Rabbit.tag += 1`과 같이 클래스 차원에서 숫자를 관리하면, 
     인스턴스가 생성될 때마다 전체 시스템의 상태를 반영한 유일한 주소를 바인딩할 수 있다.
        

---

## **# 4. [Killer Point]: 특수 메서드를 통한 객체 간 연산**

파이썬의 기본 연산자(`+`, `==`)를 우리가 설계한 클래스에 맞게 **재정의(Overriding)**하여 객체 간의 논리적 합성을 구현한다.

- **`__add__` (덧셈)**: 두 객체를 더했을 때 새로운 객체가 탄생하는 논리를 정의한다.
    
- **`__eq__` (동등성)**: 단순히 메모리 주소가 같은지가 아니라, 객체의 특정 데이터(예: 부모의 ID)가 같을 때 동일한 객체로 판정하는 비즈니스 로직을 주입한다.
