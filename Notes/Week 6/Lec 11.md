# **Program Efficiency 2**

**1. 주요 복잡도 클래스 비교 (Complexity Classes)**

가장 효율적인 것(빠른 것)부터 비효율적인 것 순서입니다 

1. O(1) **(Constant):** 입력 크기와 무관하게 항상 일정한 시간 소요.

2. O(logn) **(Logarithmic):** 데이터가 늘어나도 실행 시간은 아주 천천히 증가함 (매우 효율적) 

3. O(n) **(Linear):** 입력 크기에 비례하여 증가 (단순 `for` 루프 1개) .

4. O(nlogn) **(Log-linear):** 다음 강의(Lec 12)의 병합 정렬(Merge Sort)에서 등장

5. O(n^c) **(Polynomial):** 중첩 루프 (`for` 안에 `for`) 

6. O(c^n) **(Exponential):** 최악의 효율. 각 단계마다 재귀 호출이 2번 이상 일어나는 경우 

**2. 복잡도별 대표 알고리즘 예시**

• O(logn) **예시: 이분 탐색 (Bisection Search)**

    ◦ 탐색할 때마다 남은 데이터를 반(1/2)으로 줄여나가기 때문에 로그 시간이 걸림 

• O(2^n) **예시: 하노이의 탑 & 부분집합 (Power Set)**

    ◦ 원소가 추가될 때마다 가능한 조합의 수가 2배씩 늘어나므로 기하급수적으로 연산량이 폭발함 

**3. ★ 파이썬 기본 자료형의 시간 복잡도 (매우 중요)**

코드를 효율적으로 짜기 위해 반드시 알아야 하는 연산 속도입니다

• **리스트 (List, 길이를** n**이라 할 때)**

    ◦ O(1) **연산:** 인덱스로 접근(`L[i]`), 길이 확인(`len(L)`), 끝에 추가(`L.append()`).

    ◦ O(n) **연산:** 리스트 전체 복사(`copy`), 특정 값 탐색(`in`, `==`), 삭제(`remove`).

• **딕셔너리 (Dictionary, 길이를** n**이라 할 때)**

    ◦ O(1) **연산 (평균):** 키 검색(`index`), 값 추가(`store`), 삭제(`delete`).

    ◦ _리스트에서 값을 찾는 건_ O(n)_이지만, 딕셔너리에서 키로 찾는 건_ O(1)_이라서 압도적으로 빠릅니다!_
